Sending Karelia a Console Log

If you are troubleshooting Sandvox, Karelia may ask you to send a Console Log to help solve the issue. You only need to follow the directions below if we request such information. (Normally when reporting an issue, Sandvox includes a console log automatically, without you needing to add it manually.)

To manually send a Console Log using Tiger (Mac OS X 10.4.11):

  1. Launch the Console application (found in the Applications → Utilities folder).
  2. Make the Logs column visible by clicking the "Logs" item in the toolbar if necessary.
  3. From the list, select "console.log"
  4. Type "Sandvox" in the filter box (in the toolbar) to narrow the view down to only entries relevant to Sandvox.
  5. Optional: By looking at the date and time stamp at the start of each line, select the text relevant to your issue.
  6. From the File menu, click "Save Selection As…" and temporarily save the log somewhere convenient such as the Desktop.
  7. Begin a reply to your support e-mail requesting the console log and attach to it the log you just saved.
  8. Add any additional information to the e-mail that may be useful to us.
  9. Send the e-mail and delete the temporary log file.

To manually send a Console Log using Leopard/Snow Leopard (Mac OS X 10.5 or 10.6):

  1. Launch the Console application (found in the Applications → Utilities folder).
  2. Make the Logs column visible by clicking the "Show Log List" item in the toolbar if necessary.
  3. From the list, select "Console Messages"
  4. Type "Sandvox" in the filter box (in the toolbar) to narrow the view down to only entries relevant to Sandvox.
  5. Optional: By looking at the date and time stamp at the start of each line, select the lines relevant to your issue. Click on the first line to include and shift-click on the last line to include.
  6. From the File menu, click "Save a Copy As..." (or "Save Selection As…" if you selected lines in the previous step) and temporarily save the log somewhere convenient such as the Desktop.
  7. Begin a reply to your support e-mail requesting the console log and attach to it the log you just saved.
  8. Add any additional information to the e-mail that may be useful to us.
  9. Send the e-mail and delete the temporary log file.
