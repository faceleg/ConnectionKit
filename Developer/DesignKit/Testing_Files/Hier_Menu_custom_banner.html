Blossom - Hierarchical Menu Test

This page is part of Cinderella > Mice so those two should be highlighted as the "currentParent."

Blossom should show up as the "currentPage" and be highlighted properly.

WATCH OUT FOR LONG AND DEEPLY NESTED ITEMS.  Be sure to check out the long menu items under Snow White, and Cinderalla > Tremaine Family.

The menu should position within the visible area of the page, even though it is over on the right.

Some other things to check:

→ Test in IE7 - in about half the cases the positioning of the triangle is a few pixels off there which can look a bit odd.

→ Make sure that long menu item names do not collide with the triangle that is inserted (it may take a bit of fiddling to check that for each case). Usually some padding-right is required to ensure that.

→ Make sure the .currentParent and .currentPage items do not highlight the pages further down in the hierarchy (I am using the .currentParent > a > span.in to address the currentParent without needing to turn that behaviour off in lower levels, as this is not IE6 compatible, I am going the classical route of setting .currentPage span.in and overriding that for lower levels using .currentPage li span.in.

Sidenote: Generally it has proven very useful to do as much styling as possible on the span.in level when styling the menus, as the span.in exists in _all_ items (e.g. without link) but the span.in does _not_ contain the lower submenu levels. This makes it the best DOM element to address for menu highlightling etc. 

→ The JS inserts the .selected class for the menu items in the current flow of navigation. I usually use that for highlighting (say by using something like a:hover span.in, .selected span.in)
